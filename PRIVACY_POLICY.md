## 4. Data Security

### 4.1 Security Measures

We implement:
- 256-bit SSL/TLS encryption
- Secure password hashing (bcrypt)
- Rate limiting (10 requests/minute)
- Input validation and sanitization
- SQL injection prevention (Prisma ORM)
- XSS protection (DOMPurify)
- CSRF tokens on all forms
- Security headers (CSP, HSTS)

### 4.2 Data Storage

- Primary database: PostgreSQL (encrypted at rest)
- Backups: Encrypted and stored securely
- Session data: Encrypted cookies
- Sensitive data: Tokenized and hashed

### 4.3 Limitations

While we employ security measures:
- No system is 100% secure
- Transmission risks exist
- User responsibility to protect passwords
- We are not liable for breaches beyond our control

### 4.4 Breach Notification

If we discover a breach:
- We will notify you within 30 days
- Contact method: Email to registered address
- Information about the breach
- Steps you should take
- Our remediation efforts
